## Why NSchema?

<CardGrid stagger>
  <Card title="Declarative, not migrations" icon="document">
    You write the *final shape* of your schema in SQL-flavoured DDL — no `ALTER`
    steps, no migration files to number and order. NSchema diffs it against the
    live database and computes the plan.
  </Card>
  <Card title="Plan before you apply" icon="approve-check">
    Every run previews the exact changes first, Terraform-style. Save a plan and
    apply that exact file later, so what you reviewed is what runs.
  </Card>
  <Card title="Safe by default" icon="warning">
    Destructive changes error out unless you opt in. Drift detection, offline
    planning against persisted state, and idempotent deployment scripts round it
    out.
  </Card>
  <Card title="Adopt any database" icon="seti:db">
    Point `import` at an existing PostgreSQL database and NSchema writes your
    schema out as DDL, ready to check in and manage from there.
  </Card>
</CardGrid>
